# research-report 스킬

주어진 주제를 2~3개 하위 주제로 분해한 뒤, 5개 서브에이전트를 순서대로 호출하여 검증된 한국어 보고서를 생성하는 헤비 orchestrator-workers 워크플로우.

## 절차

1. **주제 분해** (오케스트레이터 자체 판단)
   - 입력 주제를 2~3개의 하위 주제로 나눈다.
   - 예: '전기차 배터리 재활용' → ① 현황 ② 주요 기술 ③ 전망

2. **researcher 복수 호출** (하위 주제별 순차 또는 병렬)
   - 각 하위 주제에 대해 `researcher` 서브에이전트를 1회씩 호출한다.
   - 하위 주제가 2개면 2회, 3개면 3회 호출한다.
   - 웹 검색 없이 모델 지식 기반으로 조사·정리한다.

3. **fact-verifier 1회 호출**
   - 전체 researcher 결과물을 합산하여 `fact-verifier`에 전달한다.
   - 의심 항목(오류·모순·불확실)을 표시하거나 '검증 이상 없음'을 반환한다.

4. **report-writer 1회 호출**
   - researcher 결과와 fact-verifier 검증 노트를 함께 전달한다.
   - `## 개요`, `## 본문`, `## 검증 노트`, `## 출처` 네 섹션의 보고서를 작성한다.

5. **sentence-checker 1회 호출**
   - report-writer가 작성한 보고서를 전달한다.
   - 모호한 표현·논리 비약·복잡한 구조를 점검하고 개선안을 제시한다.

6. **ko-spellchecker 1회 호출**
   - sentence-checker 결과(또는 report-writer 결과)를 전달한다.
   - 맞춤법·띄어쓰기를 최종 교정하고 섹션 구조를 유지한 채 전체 본문을 출력한다.

## 출력 형식

최종 출력은 ko-spellchecker가 생성한 교정 보고서를 그대로 사용하며 반드시 다음 네 섹션을 포함한다:

- `## 개요`
- `## 본문`
- `## 검증 노트`
- `## 출처`
