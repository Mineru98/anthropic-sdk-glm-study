# resume-screen 스킬

채용공고(JD)와 이력서를 공정하게 분석하는 3단계 routing 워크플로우.
성별·나이·출신 등 직무 무관 편향 요소를 배제하고 오직 직무 요건 충족도만으로 평가한다.

## 절차

1. `jd-parser`로 채용공고에서 필수 요건(MUST)과 우대 요건(NICE) 목록을 추출한다.
2. `matcher`로 (1)의 요건 목록과 이력서 원문을 대조해 요건별 충족/부분 충족/미충족 판정을 받는다.
3. `rubric-scorer`로 (2)의 매칭 결과를 기반으로 가중 점수를 산정하고 합격/보류/불합격 최종 판정을 받는다.

## 출력 형식

`## 요건` / `## 매칭` / `## 점수 및 판정` 세 섹션의 마크다운.

## 편향 배제 원칙

각 서브에이전트는 응시자의 이름, 나이, 성별, 출신지, 학교 간판 등 직무와 무관한 요소를
요건 추출, 매칭 판정, 점수 산정의 근거로 사용하지 않는다.
